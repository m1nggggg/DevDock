# DevDock Design Context

## Visual Direction
DevDock should feel like a clean developer command surface: quiet by default, polished in interaction, and focused on getting users to the right tool quickly.

## Theme
Light theme for daytime team usage on laptops and office monitors.

## Color Strategy
Restrained tinted neutrals with one cool accent for focus and primary actions.

## Layout
- Header with concise product name and purpose.
- Prominent search input.
- Compact responsive tool list.
- No heavy category system for the first version.
- Mobile layout should keep one readable column with actions aligned below each tool.

## Interaction
- Cards should be clickable and keyboard accessible.
- Hover and focus states should be visible but subtle.
- Search should filter by name and description.
